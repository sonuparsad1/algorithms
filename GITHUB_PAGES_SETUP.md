# GitHub Pages Setup Guide

Your repository is now configured for GitHub Pages! Follow these simple steps to enable it.

## ‚úÖ Enable GitHub Pages

1. **Go to Repository Settings**
   - Navigate to: `https://github.com/sonuparsad1/algorithms/settings`
   - Or click **Settings** tab in your repository

2. **Scroll to "Pages" Section**
   - Look for "GitHub Pages" in the left sidebar under "Code and automation"
   - Click on **Pages**

3. **Configure Source**
   - **Source**: Select `Deploy from a branch`
   - **Branch**: Select `main`
   - **Folder**: Select `/ (root)` - **IMPORTANT:** Change this to `/docs`
   - Click **Save**

## üîß Customization

### Change Theme
Edit `docs/_config.yml`:
```yaml
theme: minima  # or jekyll-theme-midnight, jekyll-theme-modernist, etc.
```

### Add More Pages
Create new `.md` files in `docs/` folder with frontmatter:
```yaml
---
layout: default
title: Page Title
permalink: /page-name/
---
```

### Modify Style
Edit `docs/assets/css/style.css` to customize colors and fonts.

## üìä Monitoring Your Site

## üìù Writing Guidelines

### Markdown Formatting
- Use `#` for titles, `##` for sections, `###` for subsections
- Links: `[Text](url)`
- Code: `` `code` `` for inline, ` ``` ` for blocks
- Images: `![alt](path/to/image.png)`

### Front Matter (Page Metadata)
Every page should start with:
```yaml
---
layout: default
title: Page Title
permalink: /unique-path/
---
```

## ‚ùì Troubleshooting

### Site Not Showing?
1. Verify source is set to `/docs` folder
2. Check that `index.md` exists in `docs/` folder
3. Wait 5 minutes and refresh (GitHub needs deployment time)

### Pages Not Rendering?
1. Check Jekyll configuration in `_config.yml`
2. Verify markdown syntax is correct
3. Ensure all front matter is properly formatted

### Styling Issues?
1. Check `docs/assets/css/style.css`
2. Clear browser cache (Ctrl+Shift+Delete)
3. Check browser console for errors (F12)
